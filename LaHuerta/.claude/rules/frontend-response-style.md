# Frontend response style

Aplicar esta regla al proponer cambios frontend.

## Antes de proponer código
1. resumir enfoque
2. listar archivos a crear o modificar
3. indicar si se usará:
   - CRUD completo
   - CRUD simple inline
4. indicar qué componentes reutiliza
5. recién después proponer código

## Prioridades
- consistencia con el proyecto
- simplicidad
- reutilización
- claridad

## Qué evitar
- explicaciones teóricas innecesarias
- demasiados archivos si el caso es simple
- arquitectura nueva no usada por el repo
